遇到的问题

  • 可能会遇到Object "netns" is unknown, try "ip help".的报错。

  1. 在CentOS6.5以前，内核不支持网络namespace，需要升级内核和iproute。CentOS6.5以后内核已支持网络namespace，只需要升级iproute即可。

  2. 默认的源中的地址已经无效，会报404，修改地址即可

  3. 安装iproute

  • 还可能遇到RTNETLINK answers: No such process的报错
    我这里在执行ip netns exec $pid ip route add default via $GATEWAY遇到的是设置容器的IP地址是没有设置掩码，默认设置为了32位，所以无法指定默认网关
