Docker中部署Zookeeper集群

上一篇汇总了Docker主机初始化及容器初始化脚本的一些内容，昨天下午在此基础上开始构建Zookeeper的镜像，今天上午结合昨天初始化容器的脚本，完成了一键启动3台Zookeeper集群的脚本，在这里记录下。

Dockerfile文件

  • jdk是用的内网自己搭建的repo，因此添加了一个repo，也可以使用公共的或编译安装，去掉那一行，修改下方yum install -y jdk1.8.0_51即可。
  • centos:6.7是根据官方镜像更新、修改的

编写启动脚本

  • ZOO_ID是定义的一个参数，会从稍后的一键启动脚本中传入，start.sh会将它写到myid文件中

一键启动脚本

  • 在昨天的容器初始化脚本上做了一些修改，将一些参数在脚本中直接设定了，不再接受传入参数，具体说明如下：
  1. ZOO_ID如上所说，会传入start.sh脚本写入myid文件
  2. Zookeeper的配置文件是映射了宿主机的一个目录，三个容器共享一份配置文件
  3. 因为无法直接修改容器内的/etc/hosts文件，这里使用—add-host传入
  4. zookeeper:0.2为根据上边Dockerfile构建的镜像及tag号

补充

  • 运行脚本后会自动启动3个容器，目前没有映射data目录，所以如果Docker出现问题后重新启动Zookeeper容器，还需要创建一些目录和节点，这个需要后续改善。
  • 目前参数都是固化在脚本中，不太适合后续做自动化工作，也需要改善，但这么多的参数感觉编写脚本是个比较麻烦的事情，再看看有没有其他的解决方案。
  • 参考了一些网上的资料：

    https://github.com/wouterd/docker-zookeeper
