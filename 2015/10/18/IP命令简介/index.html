IP命令集

命令格式：ip [ OPTIONS ] OBJECT [ COMMAND [ ARGUMENTS ]]

选项（OPTIONS）

  • -V, -Version ——- 打印版本信息
  • -s, -stats, -statistics ——- 输出更多信息
  • -f, -family {inet, inet6, link} ——- 指定使用的协议族
  • -4 —- -family inet的简写
  • -6 —- -family inet6的简写
  • -0 —- -family link的简写
  • -o, -oneline —- 将输出在一行输出，用\隔开每一行
  • -r, -resolve —- 使用DNS解析

    对象（OBJECT）

  • link —- 管理网络设备，包含物理设备及虚拟设备
  • address —- 管理设备上的IPv4、IPv6地址
  • neighbour —- 管理ARP或NDISC
  • route —- 管理路由表
  • rule —- 管理路由策略
  • maddress —- 管理多播地址
  • mroute —- 管理多播路由
  • tunnel —- 管理IP隧道
    • netns —- 管理网络命名空间，在CentOS7中预装，在CentOS6中需要升级iproute

命令（COMMAND）

各个对象支持的命令不一致，几个常用的如下：

  • show ，查看，可以缩写为sh,list,lst,ls,l
  • add ,添加，可缩写为a
  • delete，删除，可缩写为del，d
  • flush，清除，可缩写为f

ARGUMENTS

不同的对象、命令支持的ARGUMENTS不一致，主要包含两个部分

  • flags
  • parameters

ip address命令实例

show命令

  • 查看ip地址

add命令

有以下Arguments：

  • dev name —-这是我们添加地址的设备的名称
  • local address（默认值） - 接口的IP地址

    地址的格式取决于所使用协议。IPv4使用点分割的四组数字、IPv6的使用由冒号分隔的四位十六进制的序列。该地址可以跟着一个斜杠和一个十进制数，使用CIDR表示法来设定网络前缀（子网掩码）。如果没有指定CIDR网络掩码，系统会指定地址（/32掩码）。

  • peer address —- 点到点接口的远端接口地址。同样该地址可以后跟斜杠和十进制数来确定网络掩码。当peer和local同时需要指定掩码时，只能分配给peer。
  • broadcast address —- 接口的广播地址。

    可以使用特殊符号“+”和“ - ”，而不指定广播地址。在这种情况下，广播地址由或设置所有接口主机位来自于一个（+）或通过设置所有的接口主机位到零（ - ）。目前在IPv4中的网络实现中使用（+）的设置即可。

  • lable name —-每个地址可以被标上标签字符串。

    为了保持与Linux的2.0网络别名兼容，这个字符串必须与设备的名称一致，或者必须以设备名后跟一个冒号前缀。 （为eth0：杜）

  • scope SCOPE_VALUE —- 地址的有效范围。
    预定义的范围值是：

    • global —-地址是全局有效。
    • site —-（仅IPv6）本地站点地址，仅对站点内部有效。
    • link —-地址连接本地，仅在这个设备上有效
    • host —-地址仅在这台主机上有效

  • 给回环网卡lo设置标准广播地址的一个IP，有效范围为本机
